================================================================================
EDGE CASES HANDLED
================================================================================

1. Concurrent Submissions: Handled via ED-BASE IdempotencyContext.
2. In-flight Overrides: Transaction isolation + SQL conditional insert ensured 
   only the first override wins (`status='applied'`), others rejected.
3. Session Revocation: Checked on every request via `require_auth` middleware.
4. Role Changes: Re-verified at service level (e.g., Override requires ADMIN).
5. Deleted Users: `decision_actors` table supports `is_anonymized` flag to 
   preserve history without breaking refs.
6. AI Without Explanation: Validated in `rationales.py` (confidence required).
7. Decision Locking: `locked` status prevents further updates in all services.
8. Input Validation: JSON size limits and type checks enforced.
9. Replay Attacks: Idempotency keys prevent duplicate processing.
10. Partial Failures: Database transactions ensure atomicity.
